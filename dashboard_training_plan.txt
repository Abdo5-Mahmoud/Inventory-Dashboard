# Comprehensive Training & Building Plan: Production-Ready Dashboard

This document serves as your roadmap for building a real-world, portfolio-ready dashboard while solidifying your knowledge of modern frontend technologies.

---

## 1. Technologies to Master (The Real-World Stack)

To thrive in a modern frontend role, you need to deeply understand the following technologies used in this stack:

### Next.js (App Router)
- **Server Components vs. Client Components:** Understand when to use which. Default to Server Components for data fetching and SEO, use Client Components only for interactivity (hooks, event listeners).
- **Routing & Layouts:** Master nested layouts (`layout.tsx`), loading states (`loading.tsx`), and error handling (`error.tsx`).
- **Data Fetching:** Learn how to fetch data on the server and pass it to client components (or hydrate React Query).
- **Search Params:** Managing state in the URL using `useSearchParams` and Next.js navigation.

### React & TypeScript
- **TypeScript:** Strict typing, interfaces vs. types, generics (especially for API responses and React Query).
- **React Patterns:** Custom hooks (separation of logic from UI), component composition, and avoiding unnecessary re-renders.
- **Suspense:** Understanding how React Suspense works with Next.js navigation and data fetching.

### TanStack Query (React Query)
- **Caching & Stale Time:** Difference between `gcTime` (cache time) and `staleTime`.
- **Query Keys:** Designing robust, object-based query keys.
- **Mutations & Invalidation:** Optimistic updates and invalidating queries after a successful POST/PUT/DELETE.
- **Pagination & Infinite Scroll:** Using `placeholderData` (formerly `keepPreviousData`) and `useInfiniteQuery`.

### Styling & UI
- **Tailwind CSS:** Building responsive layouts (Grid/Flexbox), extracting reusable classes.
- **Component Libraries / Headless UI:** (Optional but recommended) Radix UI, Shadcn UI, or Headless UI for accessible components (modals, dropdowns).

## 2. Essential Dashboard Features

A real-world dashboard needs more than just a list of products. Here are the features you should build to make it complete:

1. **Authentication & Authorization:**
   - Login page, protected routes.
   - Role-based access (e.g., Admin vs. Viewer).

2. **Advanced Data Grids / Lists:**
   - **Pagination** (You have started this).
   - **Sorting** (Clicking table headers to sort by Price, Name, etc.).
   - **Filtering** (Category dropdowns, price range sliders).
   - **Search** (Debounced search input).
   *Note: All these should sync with the URL parameters so the page is shareable.*

3. **Product Details & Management:**
   - Dynamic route (`/products/[id]`) for viewing product details.
   - Create / Edit product forms using `React Hook Form` and `Zod` for validation.
   - Delete product functionality with a confirmation modal.

4. **Analytics / Overview Page:**
   - A dashboard home page (`/`) with charts (e.g., Recharts or Chart.js).
   - KPI cards (Total Sales, Total Products, Active Users).

5. **Settings / Profile:**
   - User profile management.
   - Theme toggling (Dark/Light mode).

## 4. Daily Habit for Real-World Preparation

- **Review the Network Tab:** Always check your browser's network tab. Ensure you aren't over-fetching data.
- **Console Check:** Keep your console free of warnings (e.g., missing keys, exhaustive-deps).
- **Commit History:** Write clean, descriptive Git commit messages. (e.g., `feat(products): implement debounced search`).
